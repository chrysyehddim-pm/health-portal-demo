# GO HEALTH 智慧健康服務平台架構書 (MVP Phase 1)

**版本:** v1.2 (Latest Update: 取消 T+1 冷卻、優化文章派發與 UI 權限)
**狀態:** 開發中 (Development)

---

## 1. 專案概述 (Overview)
GO HEALTH 是一個依附於 HAPPY GO APP 內的 Webview 服務，主打「家庭共融」與「輕量健康任務」。
MVP 階段將驗證：
1. **點數雙軌制**：個人資產 (Wallet) 與群組抽獎進度 (Group Score) 脫鉤。
2. **群組互動**：以極簡的打卡與罐頭關心，降低長輩使用門檻。

---

## 2. 核心業務邏輯 (Core Business Logic) - 🌟 最新更新

### 2.1 點數追溯與群組貢獻 (取代 T+1 冷卻)
為提升成員加入時的即時成就感，**廢除新手加入當日的冷卻機制**。
* **全週追溯邏輯：** 當成員輸入邀請碼加入群組後，系統在計算群組總分 (`weekly_score`) 時，直接 `SUM` 該成員「**本週一 00:00 至當下**」的所有有效任務點數。
* **資料庫防呆：** 後端在計算時，**不需**比對成員的 `joined_at` 時間。

### 2.2 衛教文章派發機制 (1+2 混合隨機)
為降低後端追蹤「單篇已讀」的效能負擔並確保畫面新鮮感：
* **派發策略：**
  * **第 1 篇 (輕度個人化)：** 依據使用者標籤 (年齡/性別等) 隨機抽取 1 篇；若無吻合則 Fallback 至全庫隨機。
  * **第 2、3 篇 (廣泛探索)：** 自全庫隨機抽取 2 篇。
* **去重防呆 (Deduplication)：** 後端 Query 必須確保回傳的 3 篇 `Article_ID` 皆為唯一值。
* **無狀態刷新：** 不紀錄已讀 API (`navigator.sendBeacon` 作廢)。每次重新載入首頁，即重新執行隨機抽取洗牌。

### 2.3 群組專屬暱稱權限
為簡化後端關聯表複雜度，採用群組級別的個人化設定：
* **權限限制：** 僅限使用者「本人」可以編輯自己在該群組內的暱稱。他人卡片上隱藏編輯按鈕。
* **資料庫映射 (`group_nickname`)：** 預設值為 `NULL`。若為 `NULL` 則 Fallback 顯示 `users.display_name`。

## 3. UI/UX 與前端實作規範 (Frontend Specifications)

### 3.1 去標籤化成員大頭貼
為避免年齡歧視與刻板印象，嚴禁使用靜態「長輩圖檔」，統一由前端依據 API 回傳之 `gender` 欄位動態渲染 FontAwesome 6 向量圖示：
* `M` (男): `fa-solid fa-user` + `bg-blue-100` (藍底)
* `F` (女): `fa-solid fa-user` + `bg-rose-100` (粉底)
* `U` (未知/新成員): `fa-solid fa-seedling` + `bg-emerald-100` (綠底，象徵生機)

### 3.2 色彩解耦與樣式彈性 (Theming)
* **CSS 變數化：** 品牌主色系將於五月底定案。開發期請將主色 (Primary) 定義於 `tailwind.config.js`，**嚴禁 Hardcode 顏色色碼**，確保未來能一鍵無縫換色。
* **字級統一：** 任務卡片與衛教文章標題統一使用 `text-base` (16px)，輔助說明使用 `text-sm` (14px)，確保長輩閱讀體驗。

### 3.3 通用異常處理組件 (Generic State)
當發生 API 錯誤 (4xx/5xx) 或網路斷線時，全域攔截並呼叫此組件，嚴禁顯示系統白畫面：
* **變數傳遞：** `iconType` (動態圖示), `title` (錯誤主旨), `message` (詳細原因或 Error Code), `btnText` (重試動作)。

## 4. 系統整合與協定 (Integration & JSBridge)

### 4.1 虛擬路由與返回鍵攔截
* **情境：** 開啟 Bottom Sheet 或 Modal (如：關心互動、週報、健康點說明) 時。
* **實作：** 前端需呼叫 `history.pushState` 推入虛擬路由。當使用者點擊手機實體返回鍵或 iOS 側滑時，攔截 `popstate` 事件，**僅關閉 UI 元件，不可直接退出 Webview**。

### 4.2 外部連結載入
* 衛教文章點擊後，必須呼叫原生 `window.HGBridge.openInAppBrowser(url)` 接口。
* 確保使用者關閉文章後，仍停留在原本的 Webview 任務進度中，不會重新載入首頁。

---

## 5. 後端核心排程 (Cron Jobs)

1. **每日任務重置 (00:00)：** 重置 AD8 等週期性任務狀態。
2. **連續打卡結算 (23:59)：** 掃描 Active 群組，若當日有任一有效成員完成任務，則 `consecutive_days + 1`，否則歸零。
3. **T-1 週報批次運算 (02:00)：** 統計本週至昨日的群組與個人活躍指標，產出靜態 JSON 快取。
4. **週日抽獎結算 (週日 23:59)：** 結算大於 1,000 點之群組，進行亂數抽獎，發放 HG 點數後清空 `weekly_score`。
